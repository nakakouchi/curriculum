IT：(Infomation Technology)コンピュータとネットワークを利用した技術
ICT：(Infomation Communication Technology)通信技術を使って人とインターネット、人と人を繋げる技術
IoT：(Internet of Things)人を使わずモノが自動的にインターネットと繋がる技術
OS：(Operation System)コンピューターの起動やコンピューターでアプリケーションを作動させるときに重要な役割を果たす
Windows：Micrisoft社が販売しているOS。圧倒的に対応ソフトウェアが多い
macOS(Mac OS X)：Apple社のOS。映像や絵画のソフトが充実している
UNIX：UnixはOSの一種で、現存する中でもっとも古いOSのひとつ
Linux：カスタマイズすることができるOS。コンピューターの機能を追加すること
マークアップ/フロントエンド：文章を理解させるためにルールに従ったタグを記述していくこと
HTML：WEBページを作成する言語。文字の表示
CSS：文字の色、大きさ、背景の色などを指定する言語
JavaScript：動的にコンテンツを更新や、マルチメディアを管理する言語
サーバーサイド：データベースへのデータ追加や取得、計算などサーバー側の処理全般のこと
.NET(C#,VB)：NET系のアプリ開発ができる言語。
Java：WebサービスやAndroidアプリなど作成可能な言語
Java SE：Javaの基本的な機能をまとめたもの
Java EE(Jakarta EE)：Webアプリケーション開発ができる機能がセットになったプラットフォーム
Perl：1987年開発の歴史の古いスクリプト言語。コンパイルが不要
PHP：企業のWebサイトから個人ブログまで、さまざまなWebページの制作で使われている凡庸性の高い言語
Ruby：国産のプログラミング言語。Ruby on RailsはRuby専用のフレームワーク
Apache：OSに依存しないWebサーバソフトウェアの一つ。アパッチ
nginx：Apacheと同じくOSS。Apacheより高速かつ高負荷に強い。エンジンエックス
IIS：Windows環境上で仮想サーバーを稼働させるためのソフトウェア
SQL：データベースを操作するための言語
DB2：IBM社のリレーショナルデータベース管理システム
MySQL：オープンソースのリレーショナルデータベース管理システム
Maria DB：MySQLから派生したリレーショナルデータベース管理システム
Oracle DB：堅牢性の高いリレーショナルデータベースマネジメントシステム
PostgreSQL：オープンソースのリレーショナルデータベース管理システム
SQLite：ファイルベースのデータベースエンジン
SQL Server：マイクロソフトが開発しているリレーショナルデータベース管理システム
CRUD：create(生成)read(読み取り)update(更新)delete(削除)の頭文字をとった言葉。
トランザクション：コンピュータシステムにおける、永続的なデータに対する不可分な一連の処理
コミット：commitmentの略語。関わり合うことの意味
ロールバック：データを過去のものに戻す
排他制御(排他ロック)：共有ファイルを複数の人が同時に変更した場合、排他制御されていればデータの整合性が保たれる
楽観ロック：更新対象データを取得時にロックして他者が操作できないようにする方式
悲観ロック：更新対象のデータが更新前と同じであるかを確認してから更新することで、データの整合性を保つ排他制御
実行計画：実際に実行されたプログラムの概要を提示したもの
チューニング：調整すること
パフォーマンス：性能、出来栄え
スロークエリ：実行が遅いQuery
クエリログ：実行したSQLクエリの履歴が全て記録されているログ
IDE(統合開発環境):アプリケーション構築のためのソフトウェア
Eclipse：プログラミング開発を効率化するためのIDE
NetBeans：Java SE、Java EE、PHP、JavaScript開発向けの統合開発環境
SpringToolSuite：Spring Framework を使用した開発を手助けしてくれる Eclipse ベースの開発環境
DBツール：データを格納して検索や更新、登録などを行えるようにするソフトウェア
A5:SQL MK-2：高機能かつ軽量な凡庸SQL開発ツール
pgAdmin：PostgreSQLおよび関連するデータベース管理システム用のオープンソースの管理および開発プラットフォーム
DBeaver：MySQLやPostgreSQL、SQLite、Oracle、DB2などJDBCドライバーをサポートするデータベースに対応するデータベース管理ツール
MySQL Workbench：MySQLの公式サイトにてMySQL Serverと共に配布されている、データベースの設計・開発・管理を一本で実現している無料ツール
Sequel Pro：Mac用のMySQLのデータをGUI操作で管理するためのアプリケーション
TeamSQL：Windows、Mac、マルチプラットフォーム対応のSQLクライアント
PM：プロジェクトマネージャー、複数のプロジェクトを管理
PL：プロジェクトリーダー、１つのプロジェクトを管理
PG(PE)：プログラマー、コンピューターが実行するプログラムを構築
SE：システムエンジニア、顧客からの要望をまとめる
デザイナー：製品やポスター、雑誌書籍の表紙やレイアウトなど、さまざまなもののデザインを手がける
エンジニア：工学（エンジニアリング）に関する専門的な知識やスキルを持った人材
フロントエンド・エンジニア：Webサイトの閲覧や操作などのユーザーが直接見たり、触れたりする部分（フロントエンド）を開発するエンジニア
サーバーサイド・エンジニア：サーバーの設計や構築、また運用や保守などを専門に担当
バックエンド・エンジニア：システムやサイトを利用するユーザーの目には見えることのない部分を担当するエンジニア
UI：ユーザーインターフェイス（User Interface）の略称で、一般的にユーザー（利用者）と製品やサービスとのインターフェース（接点）すべてのことを意味
CUI：画像やアイコンが無く、無い文字だけでコンピュータを操作する画面のこと
GUI：コンピュータの表示・操作体系（ユーザーインターフェース）の分類の一つで、情報の提示に画像や図形を多用し、基礎的な操作の大半をマウスやタッチスクリーンなどによる画面上の位置の指示により行うことができるもの
ファイル：データやプログラムの基本単位のことで、ハードディスクなどの記憶装置に記録されたひとまとまりの情報
拡張子：任意のファイル名の後に続く「. (ピリオド)+3～4文字」の部分
ディレクトリ：コンピュータの磁気記憶媒体に記録された、その媒体上のすべてのファイルの管理情報
ルート・ディレクトリー：ファイルやディレクトリを管理する木構造全体のうち、最上位に位置しているディレクトリのこと
カレント・ディエクトリー：簡単に言えば現在作業中のフォルダのこと
パス：あるファイルについて最上位から順番にディレクトリ名を記述したもの
絶対パス：特定のファイルの位置情報を最初から最後まで記述する方法
相対パス：編集ファイルの設置場所を起点として、今いるファイルから参照ファイルまでの場所を記述する方法
LAN：企業・事業所や大学・研究機関の内部で、文書・画像・データなどの多面的な情報を処理することのできるコンピュータ ネットワーク
WAN：遠く離れたエリアとつながったネットワークのことを指す
クライアント：「依頼人」「顧客」「福祉事業家などの世話を受ける人」と意味
サーバー：サービスを提供する側のコンピューター
HTTP：ホームページ内の文字や画像などのデータを、サーバと通信相手との間で通信するための 通信規約
FTP：ファイルを転送・通信するための規格
HTTPS：Webのデータ転送に用いられるHTTPが、SSLやTSLといったプロトコル（※）によって暗号化されている状態
SSL：トランスポート層のプロトコルであり、HTTPやFTPなどのプロトコルで行われる通信を暗号化して安全に送受信するためのプロトコ
SSH：暗号化された遠隔ログインシステム
ポート、ポート番号：コンピュータで、実際に通信される出入口の番号
HTTP、FTP、SSHの格ポート番号：HTTP(80)、FTP(20/21)、SSH(22)
ファイアウォール：外部からの攻撃を阻止したり、必要な通信は通過させる
TCP：IPの上位プロトコルのトランスポート層で動作するプロトコル
IPアドレス：ネットワーク上にある機器を識別するための数値
グローバルIPアドレス：インターネットに接続する機器が使用できるIPアドレス
プライベートIPアドレス：組織内のネットワークでのみ使用できるIPアドレス
ドメイン：個々のコンピューターを識別する名称の一部
DNS：インターネット上でドメイン名を管理・運用するために開発されたシステム
VPN：何らかのネットワーク上に仮想の専用線を実現する技術
ブリッジ：同一ネットワーク内の2つのセグメントをつなぐ
プロキシサーバー：内部ネットワークからインターネット接続を行う際、高速なアクセスや安全な通信などを確保するための中継サーバ
SMTP：送信したメールを宛先のメールサーバーまで届ける技術
POP：メールをやり取りするための用語
IMAP：インターネットアクセス
セッション：通信の開始から終了までを指す
クッキー：Webサイトへのアクセスを行なった際に、サイト側が同じ人が再訪してきたときにわかるように、訪問者のデバイスに残していくデータ
キャッシュ：表示したWebページの情報を一時的にコンピュータに保存しておく機能
インデックス：索引や見出しの意味
フロー：ものなどの流れを表した言葉で、そのものがどこに流れるのが、どのように変化していくのかということを示す
タスク：「パソコンが行っている個々の作業のこと」「コンピューターが処理する仕事の最小単位」を意味
Git：ファイルのバージョン管理が簡単にできるツール
SVN：ファイルのバージョンを管理してくれる「バージョン管理システム」
デグレ：プログラム修正による性能劣化・品質劣化
マージツール：複数のブランチの変更点を、一つのブランチに統合し、まとめる操作のことを指す
